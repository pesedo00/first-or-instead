Outcome

For teachers and parents

This tool doesn't decide whether a student can use AI. Given a task, it shows which mental operation that task should build, what gets lost when it's delegated, and how to rephrase the request to AI so it helps without replacing. The underlying thesis: delegating execution is acceptable, delegating thought is not. The difference lies in what happens before the request (an independent attempt) and after it (a check that the student genuinely possesses what was produced).

Why it doesn't use a language model

The tool never queries any AI. Asking a model to judge the use of a model would delegate the very judgment the tool is meant to build. The entire logic is a decision tree written in the open in this page's source: anyone can read it, check it, and challenge it. The table below sets it out in full.

Why it doesn't verify the stated attempt

When a student states they got stuck at a specific point, the tool asks them to write it down, and it can't check whether that's true. This limit is stated to the student at the exact moment it matters, in these words: you can write anything and move on, the only person who loses out is you. A tool that pretended to verify would teach students how to game verification. Responsibility stays where it actually sits, and the tool says so.

Where it connects to existing rules

Article 4 of the EU AI Act requires providers and professional deployers to ensure a sufficient level of AI literacy among the people acting on their behalf. That obligation already exists. The minimum content doesn't: no text says what a student should be able to tell apart. This tool is an attempt to write that content, applied to real tasks.

Article 50(2) of the same regulation requires providers of systems that generate synthetic text to mark the output in a machine-readable format. That obligation falls on the provider and concerns the content produced. One level stays uncovered, one no marking can reach: the individual student's choice, in the evening, in front of the homework. This tool works at that level.

The Italian Ministry of Education's guidelines, section 4.4 (Awareness and Responsibility), treat autonomous AI use by students as a training question: build awareness and critical judgment, learn to recognize the systems' limits and errors. No obligation, no penalty. This tool stays inside that space and accepts its boundaries.

A last note on the European framework. A student opening a chatbot to do homework is using the system for a personal activity, outside any professional context. The regulation's definition of deployer explicitly excludes this case. The regulation's obligations therefore don't reach the student: they apply to whoever provides the system and to whoever uses it in a professional or institutional setting. The student's individual choice stays outside that scope, which is why a tool like this one can only propose a criterion, never enforce it.
